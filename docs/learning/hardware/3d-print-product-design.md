## 1. 产品设计 vs 工程设计：核心区别

### 一句话总结

> **工程设计回答"怎么造出来"，产品设计回答"为什么想要它"。**

### 工程图 vs 产品概念图

| 维度 | 工程图 | 产品概念图 |
|------|--------|-----------|
| **目的** | 给制造者看，精确制造 | 给用户/投资人看，传达愿景 |
| **内容** | 尺寸标注、公差、材料、截面 | 外观、颜色、材质、使用场景 |
| **视角** | 正交投影（前/侧/顶）| 透视/等轴测，通常3/4视角 |
| **背景** | 白底或无背景 | 生活场景、渐变背景、环境光 |
| **颜色** | 单色线框或灰色实体 | 真实材质、品牌配色 |
| **标注** | 尺寸+公差 | 功能亮点callout |
| **工具** | CAD（Fusion 360, SolidWorks）| 渲染器（Blender, KeyShot）|
| **文件** | .step, .dwg, .pdf | .png, .jpg, .mp4 |

### 什么是"好看的"3D打印产品设计？

好看 ≠ 好看的3D模型。好看 = **好的设计 + 好的展示**。

**好的设计特征：**
- **圆角（fillet/chamfer）**：所有锐边倒角，至少0.5mm。这是最低成本提升质感的方式
- **分型线设计**：零件接缝在视觉上有意义（跟着产品曲面走，而不是随便切）
- **统一的设计语言**：圆角半径统一、缝隙宽度统一、螺丝孔位置对称
- **颜色和材质**：不是全部一个颜色。双色/多色打印，或者打印后喷漆/电镀
- **功能可视化**：按钮、接口、指示灯等功能元素要突出

**好的展示特征：**
- 专业渲染（不是截屏！）
- 多角度展示（至少：3/4正面、3/4背面、正面、侧面）
- 场景图（产品在使用环境中）
- 爆炸图（展示内部结构和装配关系）
- 尺寸对比（手持图或放在已知物体旁边）

### Prusa / Bambu Lab / Creality 怎么做产品展示？

**Bambu Lab（拓竹）—— 行业标杆：**
- 产品页风格明显模仿Apple：大图、少字、滚动动画
- 使用**专业产品摄影 + CAD渲染混合**
- 渲染图：KeyShot或Blender Cycles级别的PBR渲染
- 爆炸图：半透明外壳 + 内部组件高亮，不是传统爆炸图，而是"X光透视"效果
- 配色：深色背景 + 产品彩色高亮
- 动画：滚动触发的3D动画（WebGL或预渲染视频）
- **关键：他们的产品本身设计就好**——圆角、配色、分型线都经过工业设计

**Prusa：**
- 更偏"工程师审美"，但也在进步
- 产品图：白底渲染 + 使用场景照片
- 爆炸图：传统工程爆炸图风格，零件编号标注
- 相比Bambu更"实在"，展示细节多于展示feel

**Creality：**
- 早期很粗糙，近年追赶Bambu风格
- 产品页开始用大渲染图
- 但设计语言不统一，经常能感觉到"在模仿但没学到精髓"

**我们能学到什么：**
1. 产品图的第一印象由渲染质量决定
2. 深色/渐变背景比白底更高级
3. 少即是多——一张好的3/4视角渲染 > 十张截屏
4. 模型本身要"设计过"——圆角、对称、统一

## 2. 产品概念展示方法

### 三种图的区别和用途

#### 概念图（Concept Rendering）
- **用途**：传达产品愿景，给自己/团队/投资人看"最终要做成什么样"
- **特征**：完美的材质、光影、颜色；可以包含尚未实现的功能
- **精度要求**：外观准确，内部结构可以省略
- **何时做**：项目初期，确定设计方向
- **示例**：Bambu Lab产品页上的英雄图（Hero Shot）

#### 工程图（Engineering Drawing）
- **用途**：指导制造和装配
- **特征**：正交视图、尺寸标注、截面图、BOM表
- **精度要求**：100%精确
- **何时做**：设计定稿后，进入制造前
- **示例**：Prusa的装配手册

#### 场景图（Lifestyle/Context Shot）
- **用途**：展示产品在真实生活中的样子
- **特征**：产品放在桌面/手中/使用环境中
- **精度要求**：外观准确，环境可以合成
- **何时做**：产品宣传、众筹页面
- **示例**：苹果产品图——总是有人在用它

### 爆炸图的正确做法

**什么是爆炸图（Exploded View）？**

把一个装配体的零件沿装配方向拉开，展示每个零件的位置和装配关系。

**专业标准：**

1. **爆炸方向**：沿装配轴线方向拉开，不能随便散开
   - 螺丝→沿螺丝轴线方向拉出
   - 盖板→沿开盖方向拉出
   - 内部零件→按装配顺序从外到内排列

2. **爆炸距离**：均匀且适度
   - 零件间距统一（或按比例递增）
   - 太近看不清关系，太远看不出是一个产品

3. **引导线**：可选，连接零件与其装配位置的虚线
   - 专业工程爆炸图必须有
   - 产品概念爆炸图可以不要（更简洁）

4. **编号和标注**：工程版需要，概念版不需要

5. **视角**：通常等轴测（isometric）或轻微透视的3/4视角

6. **分层**：
   - 外壳层（最外）
   - 电子层（PCB、电池等）
   - 结构层（内部支架）
   - 核心层（最内）

**现代爆炸图趋势（Bambu Lab风格）：**
- 不用传统爆炸，用**半透明外壳**展示内部
- 或者**剖面图**（cutaway）：切掉一半外壳
- 或者**动画爆炸**：网页上滚动触发拆解动画
- 渲染质量和产品图一样高

### 在Blender中做爆炸图的方法

```
1. 导入所有零件STL
2. 将每个零件设为独立对象
3. 为每个零件添加关键帧动画：
   - 帧1：装配位置
   - 帧30：爆炸位置（沿装配轴平移）
4. 选择爆炸帧渲染
5. 或者渲染动画（装配→爆炸过程）
```

### 案例10：Playdate掌机
- **链接**：play.date

- **展示方式**：超精致的产品摄影+动画，黄色在灰白中跳出
- **启发**：一个小而独特的设计元素可以定义整个产品

### 从案例中提炼的规律

1. **顶级品牌都不用截屏**——都是专业渲染或摄影
2. **深色背景 > 白底**（高级感），但Amazon要求白底
3. **场景图必须有**——产品在使用环境中才有意义
4. **爆炸图正在被"X光"和"剖面"取代**——更现代
5. **一个颜色亮点**——不是全彩虹，是一个品牌色在中性色中跳出
6. **3/4视角是标准**——不是正面死板地看
7. **手持图建立尺度感**——尤其是小产品

## 4. 适合我们的工具链

### 约束回顾
- macOS arm64 (Apple Silicon)
- 命令行自动化（AI团队需要脚本调用）
- 免费/开源优先
- 专业级渲染效果

### 🏆 推荐工具链

```
建模层：OpenSCAD (工程/参数化) + Blender (外观/有机形状)
   ↓
渲染层：Blender Cycles (命令行批量渲染)
   ↓
后处理：ImageMagick (裁切/拼图/加文字)
   ↓
场景合成：Blender (HDRI背景/桌面场景)
```

### 详细工具说明

#### 3D建模：分两条路

**工程建模 → OpenSCAD（已有）**
- 适合：参数化外壳、精确尺寸零件、卡扣结构
- 输出：STL/3MF给打印机
- 自动化：✅ 完美，命令行 `openscad -o output.stl input.scad`
- **限制：不适合做有机曲面、圆滑外观**

**产品/外观建模 → Blender**
- 适合：倒角、曲面、复杂外观
- 自动化：✅ `blender --background --python script.py`
- 可以导入OpenSCAD的STL，然后在Blender中添加圆角、材质
- **这是关键的"从工程到产品"的桥梁**

**另一个选择：FreeCAD + CadQuery**
- CadQuery是Python-based的参数化CAD，比OpenSCAD更强
- 安装：`pip install cadquery`
- 支持圆角（fillet）、倒角（chamfer）等OpenSCAD不擅长的操作
- 自动化：✅ 纯Python脚本
- **推荐作为OpenSCAD的升级路径**

```python
# CadQuery示例：带圆角的盒子
import cadquery as cq
result = cq.Workplane("XY").box(60, 40, 20).edges("|Z").fillet(3)
cq.exporters.export(result, "box.step")
```

#### 渲染：STL → 漂亮的产品图

**Blender Cycles（首选）**

```bash
# 命令行渲染
blender --background scene.blend --python render_script.py --render-output //output_ --render-frame 1

# render_script.py 核心内容
import bpy
# 设置渲染引擎
bpy.context.scene.render.engine = 'CYCLES'
bpy.context.scene.cycles.samples = 256
bpy.context.scene.render.resolution_x = 1920
bpy.context.scene.render.resolution_y = 1080
# 导入STL
bpy.ops.import_mesh.stl(filepath="model.stl")
# 添加材质、灯光、相机...
bpy.ops.render.render(write_still=True)
```

**一键渲染脚本思路：**

```bash
#!/bin/bash
# render_product.sh - STL → 产品图
# 用法: ./render_product.sh model.stl output.png

BLENDER="/Applications/Blender.app/Contents/MacOS/Blender"
$BLENDER --background template.blend \
  --python import_and_render.py \
  -- "$1" "$2"
```

**关键：做一个Blender模板文件（template.blend）**
- 预设好：相机角度（3/4视角）、三点灯光、HDRI环境、地面反射
- 脚本只需要：导入STL → 自动居中 → 添加材质 → 渲染
- 这样每次渲染只需要一个命令

#### 场景合成：产品放到真实环境中

**方法1：Blender HDRI（推荐）**
- 下载HDRI环境贴图（polyhaven.com，免费CC0）
- 在Blender中设为世界背景
- 产品自动获得真实环境光和反射
- 加一个平面作为"桌面"，产品放上去

**方法2：合成（如果有实拍背景）**
- 在Blender中渲染产品（透明背景）
- 用ImageMagick或Python PIL合成到照片上
- 需要匹配光线方向

**方法3：AI生成场景（最新趋势）**
- 渲染产品透明底PNG
- 用ComfyUI/Stable Diffusion的inpainting放到AI生成的场景中
- 可以自动化，但质量不稳定

#### 爆炸图：专业装配展示

**在Blender中做爆炸图：**

```python
# explode.py - Blender脚本
import bpy
import mathutils

# 假设每个零件是独立对象
# 从装配中心向外爆炸
center = mathutils.Vector((0, 0, 0))
explode_factor = 2.0  # 爆炸系数

for obj in bpy.data.objects:
    if obj.type == 'MESH':
        direction = obj.location - center
        obj.location = obj.location + direction * explode_factor
```

**更精确的方法：手动定义每个零件的爆炸方向和距离**

```python
# 爆炸配置
explode_config = {
    "top_shell": {"axis": "Z", "distance": 30},
    "pcb": {"axis": "Z", "distance": 15},
    "battery": {"axis": "Z", "distance": 5},
    "bottom_shell": {"axis": "Z", "distance": -10},
    "screws": {"axis": "Z", "distance": -20},
}
```

#### 后处理：ImageMagick

```bash
# 添加品牌水印
convert render.png -gravity SouthEast \

# 拼接多角度图
montage front.png side.png back.png top.png \

```

### 完整自动化Pipeline

```
OpenSCAD/CadQuery → STL/STEP
       ↓
Blender Python脚本：
  1. 导入模型
  2. 自动添加材质（PBR：颜色+粗糙度+法线）
  3. 自动布置相机（多角度）
  4. 自动渲染（Cycles, 256 samples）
  5. 自动导出：hero.png, side.png, back.png, exploded.png
       ↓
ImageMagick后处理：
  裁切、拼图、加水印、加背景
       ↓
输出：产品展示图套装
```

## 5. 3D打印结构设计要点

### 卡扣设计（Snap-Fit）最佳实践

卡扣是3D打印外壳最常用的连接方式，替代螺丝。

#### 悬臂卡扣（Cantilever Snap）—— 最常用

```
        ┌──┐
        │  │← 卡钩
        │  │
   ─────┘  │← 悬臂（弹性梁）
            │
   ─────────┘← 基座（固定端）
```

**关键参数：**
- **悬臂长度**：越长越容易弯曲装配，推荐 > 5倍壁厚
- **悬臂厚度**：1.2-2.0mm（PLA），太薄会断，太厚扣不进去
- **卡钩深度（undercut）**：0.3-0.8mm，太深拆不下来
- **卡钩角度**：装入面30-45°（容易装），拔出面90°（不容易掉）
- **基座圆角**：悬臂根部必须有圆角（R ≥ 0.5mm），否则应力集中会断

**PLA卡扣注意：**
- PLA脆性大，弹性差，卡扣不能反复拆装
- 最大允许应变：~1-2%（PETG可以3-4%，TPU可以更多）
- 建议：第一次装入后就不拆了，或者设计成螺丝辅助
- **打印方向关键**：悬臂的层线方向不能垂直于弯曲方向，否则层间分离断裂

**PETG卡扣（推荐）：**
- 弹性更好，耐疲劳
- 卡扣可以反复拆装10-20次
- 壁厚可以减到1.0mm

#### 环形卡扣（Annular Snap）

用于圆柱形零件（如瓶盖）：
- 整圈卡扣需要更大装配力
- 可以做断开的环（几段弧线），减小装配力
- 适合电池仓盖等应用

### 壁厚规则

| 参数 | PLA | PETG | ABS |
|------|-----|------|-----|
| **最小壁厚** | 1.2mm (3线宽) | 1.2mm | 1.0mm |
| **推荐壁厚** | 1.6-2.0mm | 1.6-2.0mm | 1.6mm |
| **最大无支撑壁厚** | 无限制 | 无限制 | 无限制 |
| **悬挑（overhang）** | ≤45° | ≤50° | ≤45° |
| **桥接（bridge）** | ≤15mm | ≤20mm | ≤10mm |

**实用规则：**
- 壁厚 = N × 线宽（N=3或4最佳，0.4mm喷嘴 → 1.2mm或1.6mm）
- 均匀壁厚：避免厚薄突变（会翘曲）
- 加强筋（rib）比加厚更好：高度=3倍壁厚，厚度=0.5-0.75倍壁厚

### 公差和配合

**3D打印公差现实（FDM/拓竹P2S级别）：**

| 配合类型 | 间隙（单边）| 用途 |
|---------|-----------|------|
| 紧配合（Press fit）| 0.0-0.1mm | 轴承座、永久连接 |
| 过渡配合 | 0.1-0.2mm | 滑动但不松动 |
| 间隙配合 | 0.2-0.3mm | 容易装拆 |
| 活动配合 | 0.3-0.5mm | 旋转、滑动运动 |

**实用建议：**
- **先打测试件！** 每台打印机、每种材料的公差都不同
- 标准测试：打一个阶梯孔测试件（0.1mm递增）
- 拓竹P2S精度很好，通常0.15mm间隙就够

### 多零件配合设计

1. **定位特征**：用凸台/凹槽定位，不要靠螺丝定位
2. **装配顺序**：设计时就考虑从下往上装配
3. **干涉检查**：OpenSCAD中用`difference()`检查
4. **导向**：装配入口做倒角/锥度，引导对齐
5. **公差补偿**：关键配合面单边加0.15mm间隙

### 材料特性对比

| 特性 | PLA | PETG | ABS |
|------|-----|------|-----|
| **打印难度** | ⭐ 最简单 | ⭐⭐ | ⭐⭐⭐ 需要密封箱 |
| **强度** | 中等（刚且脆）| 好（有韧性）| 好（韧性好）|
| **耐温** | ~55°C | ~75°C | ~100°C |
| **卡扣适用性** | ❌ 差，易断 | ✅ 好 | ✅ 好 |
| **表面质量** | ⭐⭐⭐⭐ | ⭐⭐⭐ 拉丝 | ⭐⭐⭐ |
| **后处理** | 打磨+喷漆 | 打磨+喷漆 | 丙酮抛光！ |
| **胶水** | 502 | 502/专用 | 丙酮焊接 |
| **颜色选择** | 最多 | 较多 | 较多 |
| **气味** | 几乎无 | 轻微 | 明显 |
| **适合场景** | 原型、展示件 | 功能件、外壳 | 高温/机械件 |

**蛋蛋总经理的选择建议：**
- **展示用原型** → PLA（颜色好、表面光滑）
- **功能外壳（需要卡扣）** → PETG
- **需要后处理光滑表面** → ABS（丙酮蒸汽抛光）
- **拓竹P2S** → PLA和PETG都很好打，ABS需要密封罩

### 支撑设计

**设计原则：尽量不用支撑！**

1. **45°规则**：所有悬挑角度 ≤ 45°
2. **分件**：一个复杂零件拆成两个简单零件（各自不需要支撑）
3. **打印方向**：选择需要最少支撑的方向
4. **桥接**：短距离水平悬空可以桥接（≤15mm for PLA）
5. **自支撑形状**：用45°斜面代替90°悬空
6. **支撑接触面**：如果必须用支撑，把支撑面设计在不可见的内部

## 6. 从"能量核心"到"完整体"的展示方法

> 核心问题：怎么让人看一个小零件，就能想象出完整的产品？

### 方法论

这个问题的本质是**"部分与整体的关系可视化"**。

### 方法1：Ghost/轮廓展示法（推荐）

```
┌─────────────────────────┐
│                         │
│    ╭─ ─ ─ ─ ─ ─ ╮     │   ← 完整体的半透明轮廓
│    ╎  ┌───────┐  ╎     │
│    ╎  │ 能量  │  ╎     │   ← 实体渲染的当前零件
│    ╎  │ 核心  │  ╎     │
│    ╎  └───────┘  ╎     │
│    ╎             ╎     │
│    ╰─ ─ ─ ─ ─ ─ ╯     │
│                         │
│   "能量核心 — 第1步/共5步" │
└─────────────────────────┘
```

**实现：**
- 当前零件：正常材质、正常渲染
- 完整产品：半透明（opacity 10-20%）或线框模式
- 当前零件在完整产品中的位置高亮
- 这是最直观的方式

**参考：**
- 乐高说明书的"当前步骤"展示：已装的灰色/半透明，当前步骤的零件彩色高亮
- 汽车零配件网站：总是在汽车透明图上标注零件位置

### 方法2：进度路线图展示法

```
  ┌──┐    ┌──┐    ┌──┐    ┌──┐    ┌──┐
  │01│───→│02│───→│03│───→│04│───→│05│
  │🔋│    │📦│    │🔌│    │🖥 │    │🏠│
  │核心│    │外壳│    │接口│    │屏幕│    │完整│
  └──┘    └──┘    └──┘    └──┘    └──┘
   ★当前    下一步   待做     待做    最终目标
```

**实现：**
- 页面顶部或侧边放一个进度条
- 每个步骤是一个模块/组件
- 当前模块高亮，已完成的打勾，待做的灰色
- 最右边是完整产品的渲染图

**参考：**
- 众筹（Kickstarter）的Stretch Goals展示
- 游戏的科技树

### 方法3：乐高式包装展示法

```
┌────────────────────────────────┐
│  ┌──────────┐  ┌────────────┐ │
│  │          │  │  ★完整产品  │ │
│  │  你现在   │  │   渲染图   │ │
│  │  拿到的   │  │ （最终效果）│ │
│  │  零件包   │  │            │ │
│  └──────────┘  └────────────┘ │
│                                │
│  包含：能量核心 x1             │
│  还需要：外壳包、接口包...     │
└────────────────────────────────┘
```

**参考：**
- 乐高盒子正面：总是展示完成品
- 乐高盒子背面：展示可玩性/不同形态
- 乐高内袋：编号分包，说明书标注哪个包用于哪个步骤

### 方法4：Apple Watch表带式展示法

苹果展示手表表带的方法：
1. **单品图**：表带自身的美——材质、颜色、细节
2. **穿戴图**：表带装在手表上，手表在手腕上
3. **全家福**：所有颜色/款式排列

**转化到我们的场景：**
1. **单品图**：能量核心自身的美——灯效、材质、尺寸
2. **装配图**：能量核心装入完整产品中
3. **使用图**：完整产品在使用场景中（桌面/手中）

### 方法5：剖面/X光展示法

```
┌─────────────────────┐
│    完整产品外观      │
│                     │
│   ┌──────────────┐  │
│   │   内部结构    │  │
│   │  ┌────────┐  │  │
│   │  │★能量核心│  │  │   ← 高亮色、发光效果
│   │  └────────┘  │  │
│   │              │  │
│   └──────────────┘  │
│                     │
└─────────────────────┘
```

**实现：**
- 渲染完整产品，但外壳半透明
- 当前展示的零件用品牌色高亮+发光效果
- 像汽车广告展示发动机一样

**这是最酷、最现代的方式**，Bambu Lab就是这么做的。

### 推荐组合

对于我们的产品展示页，建议使用：

1. **Hero区域**：完整产品渲染（即使是概念，也要展示最终目标）
2. **当前进度**：Ghost展示法（方法1）—— 半透明完整体+实体当前模块
3. **模块详情**：每个模块单独的3/4渲染+功能callout
4. **进度路线图**：方法2 —— 展示从零到完整的路径
5. **场景图**：完整产品在使用场景中（即使是渲染/概念）

### 实操步骤（从今天开始）

```
Step 1: 安装Blender
  brew install --cask blender

Step 2: 制作Blender渲染模板
  - 设置相机、灯光、HDRI背景
  - 保存为template.blend

Step 3: 写一个STL→产品图的Python脚本
  - 导入STL
  - 自动居中和缩放
  - 添加基础PBR材质
  - 渲染输出

Step 4: 渲染当前零件
  - 能量核心的hero shot
  - 多角度图

Step 5: 建模/渲染完整产品概念
  - 即使很粗糙，也要有一个"完全体"的概念模型
  - 不需要精确，只需要传达"最终形态"

Step 6: 做Ghost展示图
  - 完整体半透明 + 能量核心高亮
  - 这张图就能回答"这个小东西是什么"

Step 7: 做产品页
  - 按上面的推荐组合排列
```

### 设计checklist

- [ ] 壁厚是线宽的整数倍
- [ ] 卡扣根部有圆角

- [ ] 不需要支撑（或支撑在隐藏面）
- [ ] 有定位特征（凸台/凹槽）
- [ ] 装配顺序清晰

### 展示checklist

- [ ] 有3/4视角Hero Shot

- [ ] 有爆炸图或剖面图
- [ ] 有场景/使用图

### 一行命令速查

```bash
# OpenSCAD → STL
openscad -o part.stl part.scad

# Blender渲染
blender -b template.blend -P render.py -- input.stl output.png

# CadQuery → STEP
python3 -c "import cadquery as cq; cq.exporters.export(cq.Workplane().box(10,10,10).edges().fillet(1), 'box.step')"

# ImageMagick拼图
montage *.png -tile 2x2 -geometry 800x800+5+5 combined.png

# 加深色背景
convert -size 1920x1080 gradient:'#0d1117'-'#161b22' bg.png
composite -gravity center render.png bg.png final.png
```
