# 用户体验全链路审查报告

> 审查人：蛋蛋（GM） | 日期：2026-02-24  
> 基于 landing 站点代码、平台代码及文档审查

---

## 总体评分：5.5/10

视觉设计质量高，但核心用户旅程断裂严重。用户从"看到首页"到"真正用起来"存在多处死亡点。

---

## 一、首页体验（landing/index.html）

### ✅ 优点
- 视觉设计专业，深色科技风格统一（配色、字体、动画都到位）
- 响应式处理有（移动端隐藏导航链接）
- 有 GitHub badge、Star 数等社会证明元素
- 模块卡片信息密度合理（BOM成本、难度、标签）

### ⚠️ 问题

**1. 核心叙事与实际产品脱节 — 严重**
- Hero 标题 "AI Controls the Physical World" → CTA 是 "Get Started" → 指向 app.realworldclaw.com/auth/register
- 用户期待：注册后能让 AI 控制什么东西
- 实际：注册后看到的是一个平台 dashboard，离"AI控制物理世界"还有巨大鸿沟
- **建议**：CTA 改为指向 Quickstart 文档或交互式 demo，降低期望差

**2. "Explore the Map" 链接指向空地图 — 严重**
- 无真实节点 = 空白页面 = 立即流失
- **建议**：地图未有数据前，删除此 CTA 或改为截图/视频预览

**3. 移动端导航完全隐藏 — 中等**
- `@media(max-width:768px){.nav-links{display:none}}` → 移动端无任何导航
- 无汉堡菜单、无替代导航方案
- **建议**：加汉堡菜单或底部固定 CTA

**4. 视频占位未填 — 中等**
- Hero 区域有视频占位但无真实视频
- 承诺了"真实运行视频"但未兑现

---

## 二、注册/登录流程

### ⚠️ 问题

**1. 注册后旅程不明确 — 严重**
- 用户注册后看到什么？没有 onboarding 引导
- 无"下一步做什么"的提示
- **建议**：注册后展示 3 步引导（选身份→看教程→开始操作）

**2. 角色未分流 — 中等**
- 首页有三种用户（开发者/Maker/团队）但注册流程统一
- 注册后不知道自己该看哪个页面
- **建议**：注册时选择角色，后续展示对应内容

---

## 三、开发者体验（核心用户路径）

### ✅ 优点
- API 文档（api.realworldclaw.com/docs）在线可访问
- Quickstart 文档存在且有步骤
- Python/TypeScript SDK 均可用
- OpenClaw Skill 提供了轻量入口

### ⚠️ 问题

**1. SDK → 真实设备的鸿沟 — 严重**
- SDK 文档齐全，但从"安装SDK"到"控制真实设备"之间缺乏桥梁
- 需要硬件（ESP32+传感器），大多数开发者不会立即购买
- **建议**：Module Emulator 应作为首选入口（"零硬件先体验"），当前在 tools/emulator/ 但首页未提及

**2. OpenClaw Skill 入口埋得太深 — 中等**
- Skill 是战略级入口，但在首页不够突出
- **建议**：首页加独立 section："Already using OpenClaw? Add RWC in 30 seconds"

---

## 四、Dashboard/App 体验

### ⚠️ 问题

**1. 设备页面无设备时体验差 — 中等**
- 新用户看到空列表，无引导
- **建议**：空状态加引导卡片（"添加你的第一个模块"→链接到 Quickstart）

**2. 功能页面与实际能力不匹配 — 中等**
- 存在页面路由（25个）但多数功能未实现或数据为空
- 与"零假功能"禁令存在潜在冲突

---

## 五、全链路断裂点总结

```
用户旅程中的"死亡点"：

首页 ──→ 注册 ──→ Dashboard ──→ ???
  ↑         ↑          ↑           ↑
  OK      OK但      空白页面    无下一步
         无分流     无引导      旅程断裂
                                
理想旅程：
首页 → 选身份 → 对应教程 → 模拟器体验 → 真实硬件（可选）
```

---

## 六、优先修复建议

| 优先级 | 问题 | 修复成本 | 影响 |
|--------|------|---------|------|
| P0 | 移动端无导航 | 低 | 移动用户全部流失 |
| P0 | 空地图CTA | 低 | 点击即失望 |
| P1 | 注册后无引导 | 中 | 新用户留存为零 |
| P1 | 模拟器入口缺失 | 低 | 开发者无法零成本体验 |
| P1 | Skill入口不突出 | 低 | 战略入口被埋没 |
| P2 | 视频占位 | 中 | 首页可信度降低 |
| P2 | 角色分流 | 中 | 体验泛化 |
