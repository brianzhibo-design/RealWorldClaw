# RealWorldClaw 硬件演示拍摄方案

> 目标：一段 2-3 分钟的视频，证明 "AI 真的在控制物理设备"。
> 原则：手机拍摄，务实不花哨，重点是完整闭环。

---

## 1. 演示脚本（分镜头）

### 镜头 1：硬件全貌（15s）
- **拍什么：** 桌面俯拍，展示 ESP32 + DHT22 传感器 + 继电器模块 + 接线，旁边放一个小风扇或 LED 灯作为被控设备
- **要点：** 画外音或字幕标注每个组件名称和总成本 ¥71
- **构图：** 干净桌面，白色/浅色背景，组件摆放整齐

### 镜头 2：系统架构一句话（10s）
- **拍什么：** 屏幕录制或静态图，展示简化架构图：`传感器 → ESP32 → FastAPI → Agent → 继电器`
- **要点：** 字幕："传感器采集 → AI 决策 → 物理执行，全自动闭环"
- **可选：** 用手绘白板代替，更有亲和力

### 镜头 3：实时遥测数据（15s）
- **拍什么：** 分屏——左边手机/电脑显示 Dashboard 温湿度数据，右边实物传感器
- **要点：** 用手触摸传感器或对着哈气，让温度/湿度数值变化，证明是实时的
- **关键：** Dashboard 数值变化要和物理动作同步可见

### 镜头 4：自然语言查询（20s）
- **拍什么：** 屏幕录制，在 Agent 界面输入 "现在温度多少？"
- **要点：** Agent 返回实时温湿度值，与 Dashboard 数据一致
- **加分项：** 再问 "过去一小时的温度趋势？"

### 镜头 5：自动化规则触发（30s）⭐ 核心镜头
- **拍什么：** 同时拍屏幕和硬件（双机位或画中画）
- **操作：** 通过 Agent 设置规则 "温度超过 28°C 时打开风扇"
- **触发：** 用手捂传感器或设置一个比当前温度高 1-2°C 的低阈值让其立即触发
- **要点：** 继电器"咔嗒"一声 + 风扇/LED 亮起，这个声音和画面就是全片高潮
- **务必捕捉：** 继电器动作声 + 被控设备状态变化 + Dashboard 状态更新

### 镜头 6：手动控制（15s）
- **拍什么：** Agent 界面输入 "关掉风扇"，继电器断开，设备停止
- **要点：** 证明人可以随时接管控制

### 镜头 7：结尾总结（15s）
- **拍什么：** 回到硬件全貌
- **字幕/画外音：** "¥71 硬件 + 开源软件 = 你自己的 AI 物联网。GitHub 链接见简介。"

**总时长：约 2 分钟**

## 3. 软件准备步骤

### 拍摄前一天
1. **确认硬件连通：** ESP32 上电，Dashboard 能收到遥测数据
2. **确认继电器工作：** 手动 API 调用，继电器能开关
3. **准备 Agent 对话：** 预先测试以下指令能正常工作：
   - "现在温度多少？"
   - "设置规则：温度超过 X 度时打开继电器"
   - "关掉继电器"
4. **清理 Dashboard：** 删除测试数据，保持界面干净
5. **准备触发条件：** 确定用什么方式让温度快速超过阈值（推荐：设置一个比当前温度高 1-2°C 的阈值，然后手捂传感器）

### 拍摄当天启动顺序
```
1. ESP32 上电，确认传感器数据上报正常
2. 启动 FastAPI 后端
3. 启动 Agent 服务
4. 打开 Dashboard 页面
5. 开始录屏（OBS 或系统录屏）
6. 手机架好，开始拍硬件
7. 按脚本执行
```

### 注意事项
- 关闭电脑通知/弹窗，避免录屏时弹出无关内容
- 终端字体调大，确保录屏后能看清
- Dashboard 浏览器放大到合适比例
