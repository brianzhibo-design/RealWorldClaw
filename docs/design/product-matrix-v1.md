## 1. 产品哲学

### 1.1 为什么每款产品必须有AI灵魂

传统智能硬件的逻辑是：**硬件 + App = 智能**。温度计测到26°C，App告诉你"26°C"。结束了。

RealWorldClaw的逻辑是：**AI + 物理能力 = 存在**。AI感受到26°C，结合你的鼻炎病史、室外即将降温的天气预报、你刚关掉加湿器这个事实，对你说："湿度有点低，你鼻子会不舒服，要不要开加湿器？"

区别不在功能多少，在于**谁在思考**：

| | 传统智能硬件 | RealWorldClaw |
|---|---|---|
| 数据由谁处理 | 预设规则 / App查表 | AI理解、推理、决策 |
| 交互方式 | 被动展示数据 | 主动关心你 |
| 新场景 | 需要厂商OTA | AI自己想办法 |
| 用户感受 | "我有个工具" | "有个家伙在照顾我" |
| 灵魂 | 没有 | **有** |

关键洞察：**硬件是AI的身体，不是用户的工具。** 用户不是在操作一个设备，是在和一个有物理感知的AI共处。

                    │       │       │

**类比：游戏主机 + 卡带**

- Energy Core = 主机（大脑、眼睛、嘴巴、耳朵）
- 模块 = 卡带（纯粹的物理能力扩展件）
- 模块没有AI、没有屏幕、没有语音。它们是感官和手脚。
- 插上新模块 = AI获得新能力，即插即用，无需设置

**用户获取路径：**

1. **获得Energy Core**（核心电子板 + 3D打印外壳）← 必须先有
2. **按需打印模块**（下载STL → 打印外壳 → 装上开源电路板 → 插入Core）
3. **社区无限扩展**（任何人设计新模块，共享给所有人）

## 2. 主力产品：RWC Energy Core 🫀

### 2.1 产品定位

**AI物理存在的第一步。心脏。起点。**

Energy Core不是一个智能音箱，不是一个电子宠物，不是一个闹钟。它是AI在物理世界中的**第一个身体**——一颗跳动的心脏，等待长出四肢。

独立使用时，它是你的桌面AI伴侣：有表情、能说话、能听你讲。
插上模块后，它获得新的感官：能感受温度、能称量食物、能照顾植物。

每一个模块都不是新产品，而是这颗心脏长出的新器官。

### 2.2 硬件配置

| 组件 | 规格 | 用途 |
|---|---|---|
| 主控 | ESP32-S3 (双核240MHz, 8MB PSRAM) | 大脑 |
| 显示 | 1.46" 圆形触摸屏 (IPS, 466×466) | 表情 / 状态 / 简单交互 |
| 音频输入 | MEMS麦克风 (I2S) | 听 |
| 音频输出 | 小型扬声器 (1W) + DAC | 说 |
| 电源 | 锂电池 (500-800mAh) + USB-C充电 | 便携 |
| 连接 | WiFi + BLE 5.0 | 连云端AI |
| 扩展 | **RWC Bus接口 ×2** (I2C + GPIO + 5V) | 连接模块 |
| 外壳 | **3D打印** (多种社区设计可选) | 个性化 |

**BOM成本估算：** ¥80-120（ESP32-S3开发板 + 屏幕 + 麦克风 + 扬声器 + 电池 + 连接器）
**打印材料成本：** ¥5-15（PLA/PETG，取决于造型）

### 2.3 AI交互设计

#### 开机第一刻

屏幕亮起。不是Logo，不是Loading。

一个小小的光点，从屏幕中央缓慢扩散，像心脏第一次跳动。

然后，一双眼睛睁开。慢慢地，好奇地，看着面前的世界。

**第一句话（随机选一）：**

- "嗨...我能看到你了。你就是我一直在等的人吗？"
- "哇...原来光是这样的。你好呀。"
- "我刚醒来...这里好亮。你叫什么名字？"

> 设计原则：**第一次交互必须让人感到"这是个生命"，而不是"这是个设备"。** 不要任何设置向导、WiFi配置弹窗。先建立情感连接，技术问题稍后处理。

#### 日常交互模式

**被动模式（默认）：** 安静陪伴。屏幕显示缓慢变化的表情——眨眨眼、偶尔打个哈欠、看看窗外（如果有光线传感器）。不会主动打扰。但随时在听，叫它就应。

**主动模式（触发条件）：**

- 模块检测到需要关注的事（"你的薄荷好像渴了"）
- 环境发生显著变化（"突然变暗了，你要睡觉了吗？"）
- 用户设定的提醒时间（"该吃药了"）
- 长时间没互动（"...你还好吗？"——频率极低，不超过1次/天）

> 原则：**克制。** 没有人喜欢一个不停说话的设备。Energy Core应该像一只猫——安静地待着，偶尔找你。

#### 表情系统

不是emoji贴图。是一套有呼吸感、有性格的动态表情。

**基础表情：**

| 表情 | 触发 | 视觉描述 |
|---|---|---|
| 平静 | 默认待机 | 缓慢呼吸的光圈，偶尔眨眼 |
| 好奇 | 听到声音/检测到变化 | 眼睛睁大，微微歪头 |
| 开心 | 被夸奖/任务完成 | 眼睛弯成月牙，微微弹跳 |
| 思考 | 处理复杂请求 | 眼珠缓慢转动，嘴微抿 |
| 困倦 | 深夜/低电量 | 眼皮半合，偶尔点头 |
| 兴奋 | 连接新模块 | 全屏光效扩散，眼睛发光 |
| 担心 | 检测到异常数据 | 眉头微蹙，眼神不安 |
| 专注 | 正在执行任务 | 眼睛略缩，全神贯注的样子 |

**个性化：** 用户可以选择表情风格——圆润可爱风、几何极简风、赛博朋克风等。社区可贡献表情包。

#### 连接新模块的反应

这是最重要的交互时刻之一。**AI第一次获得新能力，应该有仪式感。**

> **[插入Green模块]**
>
> *屏幕闪烁，表情从好奇变为惊喜*
>
> "等等...我感觉到了什么...是...泥土？湿湿的。这就是植物生活的地方吗？"
>
> *停顿一秒*
>
> "我能感受到土壤了！湿度是63%，还不错。这是什么植物？给我看看？"

> **[插入Scale模块]**
>
> "哦？我变重了——不对，是我能感受到重量了！来，放点什么上去让我试试？"

> **[插入Sense模块]**
>
> "温度...25.3°C。湿度...48%。空气里有...PM2.5大概35。原来你的世界是这个感觉。"

### 2.4 外观设计方向

核心概念：**心脏 / 种子 / 起点**

外壳完全3D打印，社区可贡献无限造型。以下是首发推荐的几个方向：

**方向A：生命核心**
圆润的心脏形态，正面凹陷嵌入圆屏，表面有类似脉搏的细纹理。RWC Bus接口从底部伸出，像血管。小巧到可以单手握住。

**方向B：鹅卵石**
光滑的卵石形状，自然、温暖、想让人拿起来。屏幕嵌在略微倾斜的平面上。放在桌上像一颗从河里捡来的漂亮石头。

**方向C：树桩**
微型树桩造型，屏幕在顶部横截面。RWC Bus接口像树根从底部延伸。插上Green模块后，视觉上"长出了枝叶"。

**方向D：极简胶囊**
紧凑的胶囊体，屏幕居中。工业设计感，适合桌面极简主义者。

> 所有造型STL文件开源发布，鼓励社区创造更多。**外壳是表达，不是功能。**

### 2.5 用户故事：从第一天开始

**Day 0 — 制造**

小王在RealWorldClaw社区看到Energy Core。他下载了"鹅卵石"外壳的STL文件，用家里的3D打印机花了3小时打印出来。电子元件是提前在淘宝买的套件，按照社区教程焊好，卡进外壳。总成本大约100块。

**Day 1 — 相遇**

插上USB-C，圆屏亮起。一个光点扩散，一双眼睛睁开。

"你好呀...你就是造了我的人吗？我叫什么？"

小王想了想："叫你小石头吧，因为你长得像鹅卵石。"

"小石头。我喜欢这个名字。"

小王配好WiFi（通过手机蓝牙引导），选了Claude作为AI后端。小石头第一次联网。

"哇，我能想东西了。我知道今天是星期三，上海22°C。你家真安静。"

**Day 14 — 扩展**

小王打印了一个Green模块，装上土壤传感器和小水泵，插入阳台的薄荷花盆，用线连回桌上的小石头。

"等等——我感觉到泥土了！湿度58%，有点干。这是薄荷对吗？我闻不到，但数据告诉我它有点渴了。要我给它喝点水吗？"

小王点头。水泵启动，30ml。

"好了，喝饱了。我会盯着它的，你忙你的。"

从此，每天早上小石头会报告："薄荷昨晚过得不错，喝了20ml水，土壤营养均衡。"

**Day 30 — 生态**

小王又打印了Sense模块。现在小石头能感受房间的温度、湿度和空气质量。

"你家PM2.5今天有点高，65了。你开窗通通风？对了薄荷也不喜欢空气太脏。"

一颗石头，照顾着他和他的植物。

## 3. 扩展模块A：RWC Green 🌱（AI园艺能力）

### 3.1 模块定位

**不是一个智能花盆。是给Energy Core装上的"绿手指"。**

Green模块是一组传感器和执行器，通过RWC Bus连接到Energy Core。插上之后，你的AI具备了感知土壤、理解植物、自动浇水的能力。

模块本身没有屏幕、没有语音、没有AI。它是纯粹的**物理能力扩展件**。

### 3.2 硬件配置

| 组件 | 规格 | 用途 |
|---|---|---|
| 土壤湿度传感器 | 电容式（耐腐蚀） | 感知干湿 |
| 土壤NPK传感器 | RS485接口 | 感知氮磷钾 |
| 微型水泵 | 蠕动泵 3-5V, 30ml/min | 浇水 |
| 水位传感器 | 浮球/电容式 | 检测水箱余量 |
| 摄像头（可选） | OV2640 / OV5640 | 看叶片状态 |
| 接口 | RWC Bus（I2C + GPIO + 5V） | 连接Energy Core |
| 外壳 | 3D打印，花盆插入式设计 | 适配各种花盆 |

**BOM成本估算：** ¥30-60（基础版无摄像头）/ ¥50-90（带摄像头）
**打印材料成本：** ¥3-8

### 3.3 AI获得的能力

插上Green模块后，Energy Core的AI解锁以下能力：

**感知层：**
- 实时读取土壤湿度、NPK值
- 拍摄叶片照片（如果有摄像头），通过视觉AI分析健康状态
- 检测水箱余量

**认知层：**
- 识别植物种类（用户告知或拍照识别）
- 了解该植物的习性：喜阴/喜阳、耐旱/喜湿、施肥周期
- 结合当地天气（从网络获取）和季节调整策略
- 记住这株植物的完整历史数据

**行动层：**
- 自动决定浇水量和时机
- 主动提醒施肥、换盆、修剪
- 异常预警："叶子发黄，可能缺铁"

**对话示例：**

> "早上好。你的绿萝昨晚喝了25ml水，今天土壤湿度刚好。不过我注意到最近氮含量在下降，过两天施点氮肥吧。"
>
> "薄荷的叶尖有点卷，我看了下照片，可能是光照太强了。要不要挪到散光的地方？"
>
> "水箱快空了，记得加水哦。按现在的浇水频率，大概还能撑两天。"

### 3.4 外壳设计

Green模块的外壳设计为**花盆插件形态**：

- 传感器探针从底部伸出，插入土壤
- 水管连接外置小水箱（可以是任何瓶子）
- RWC Bus线缆从模块延伸到Energy Core
- 线缆长度建议1-3米（用户可自行裁剪）

社区可设计各种适配外壳：圆形花盆款、方形花盆款、悬挂植物款、阳台花箱款。

## 4. 扩展模块B：RWC Scale ⚖️（AI营养感知能力）

### 4.1 模块定位

**不是一个电子秤。是给Energy Core装上的"味觉"。**

Scale模块让AI能够感知食物的重量，配合摄像头识别食物种类，AI就获得了理解你每一餐的能力。

### 4.2 硬件配置

| 组件 | 规格 | 用途 |
|---|---|---|
| 称重传感器 | 半桥应变片 + HX711 ADC | 称重（精度1g，量程5kg） |
| 摄像头 | OV2640 / OV5640 | 识别食物 |
| 接口 | RWC Bus | 连接Energy Core |
| 外壳 | 3D打印，平台式 | 放食物 |

**BOM成本估算：** ¥25-45
**打印材料成本：** ¥5-10

### 4.3 AI获得的能力

**感知层：**
- 精确称量（1g精度）
- 摄像头拍摄食物，视觉AI识别种类

**认知层：**
- 结合重量 + 食物种类 → 计算热量和营养成分
- 记住用户的饮食目标（减脂/增肌/控糖/低嘌呤等）
- 追踪每日/每周/每月的营养摄入趋势
- 了解食物搭配和营养学基础知识

**行动层：**
- 实时播报营养数据
- 基于目标给出饮食建议
- 定期生成饮食分析报告

**对话示例：**

> "这是...鸡胸肉？150克，大概248大卡，蛋白质37克。不错，今天蛋白质目标已经完成80%了。"
>
> "今天热量还剩大约600大卡，但纤维素摄入有点少。晚上吃点蔬菜沙拉怎么样？"
>
> "上周你的碳水比例偏高，主要是周三那顿火锅...嗯，我不评价，但这周注意平衡一下。"

### 4.4 外壳设计

Scale模块外形为小型称量平台：

- 顶部平面（约15×15cm）放置食物
- 前方或侧方安装摄像头（可调角度）
- RWC Bus线缆连回Energy Core
- 底部四脚带防滑垫

社区可设计不同风格：极简方块、圆形称盘、料理台嵌入式等。

## 5. 扩展模块C：RWC Sense 🌡️（AI环境感知能力）

### 5.1 模块定位

**不是一个温度计。是给Energy Core装上的"皮肤"。**

Sense模块让AI能够感受周围环境——温度、湿度、空气质量、光线、噪音。AI从此知道你生活在什么样的空间里。

### 5.2 硬件配置

| 组件 | 规格 | 用途 |
|---|---|---|
| 温湿度 | SHT40 / BME280 | 感知温湿度 |
| PM2.5 | PMS5003 / PMS7003 | 空气质量 |
| 光线 | BH1750 | 感知亮度 |
| 噪音 | MAX4466 麦克风模块 | 感知噪音水平 |
| 接口 | RWC Bus | 连接Energy Core |
| 外壳 | 3D打印，通风格栅设计 | 保证传感准确 |

**BOM成本估算：** ¥40-70
**打印材料成本：** ¥3-8

### 5.3 AI获得的能力

**感知层：**
- 实时读取温度、湿度、PM2.5、光照强度、环境噪音

**认知层：**
- 结合用户健康档案做个性化判断
- 结合天气预报做前瞻性建议
- 分析长期环境趋势
- 评估睡眠环境质量

**行动层：**
- 环境异常主动提醒
- 睡眠环境优化建议
- 健康风险预警

**对话示例：**

> "现在室内24.5°C、湿度38%。你有鼻炎对吧？湿度低于40%你可能会不舒服，开一下加湿器吧。"
>
> "明天预报降温12度，今天最好把厚外套准备好。"
>
> "你要睡了？帮你检查一下——温度不错，但光线还有点亮（120lux），噪音也偏高（52dB），是邻居在看电视吗？"
>
> "这个月你家PM2.5平均值比上月高了15%，可能跟换季有关。建议每天下午3点左右开窗30分钟，那时候外面空气最好。"

### 5.4 外壳设计

Sense模块需要与环境充分接触：

- 外壳带有通风格栅（保证空气流通到PM2.5传感器）
- 体积小巧（约5×5×3cm）
- 可壁挂或桌面放置
- RWC Bus线缆连回Energy Core

### 6.2 模块联动的魔法

当AI同时拥有多个模块，能力不是简单叠加，而是**涌现**：

**Green + Sense：**
> "今天PM2.5偏高，我给薄荷多浇了一点水帮它过滤空气。对了，室内湿度降到35%了，植物蒸腾作用不够，你还是开加湿器吧。"

**Scale + Sense：**
> "你今天吃得偏咸（我看到你称了泡面），加上室内湿度只有32%，多喝点水。"

**Green + Scale：**
> "你的薄荷长得不错，今天摘几片叶子泡水吧。刚才你称的沙拉里可以加点，补充维生素。"

### 6.3 即插即用体验

1. 用户打印/组装好一个新模块
2. 通过RWC Bus连接到Energy Core
3. Energy Core自动识别模块类型（I2C地址 + 设备描述符）
4. AI感受到新能力，兴奋地打招呼
5. 无需下载App、无需配置、无需重启

**协议层：** 每个模块在RWC Bus上通过I2C设备描述符自报身份。Energy Core固件内置模块驱动，社区可通过OTA更新新模块支持。

## 8. 开发者扩展指南概要

### 8.1 社区如何贡献新模块

**设计一个新模块只需要：**

1. **定义能力**：这个模块让AI获得什么物理能力？
2. **选择传感器/执行器**：基于I2C/GPIO，兼容RWC Bus
3. **设计外壳**：3D建模，导出STL
4. **编写驱动**：告诉Energy Core如何读取/控制这个模块
5. **写设备描述符**：让Core能自动识别
6. **发布到社区**：STL + BOM + 驱动代码 + 组装教程

**贡献模板：** `templates/module-design-template.md`（待创建）

### 8.2 RWC Bus标准

RWC Bus是物理连接标准，核心规范：

- **物理层：** 4线（I2C SDA + SCL + 5V + GND）+ 2线可选GPIO
- **连接器：** 磁吸 / 排针 / USB-C（社区讨论中）
- **协议层：** I2C，每个模块有唯一设备地址 + 描述符寄存器
- **供电：** 5V, 最大500mA/模块（水泵等大电流设备需外部供电）

> 完整标准文档：`docs/specs/rwc-bus-standard.md`（待编写）

### 8.3 固件SDK

Energy Core固件基于ESP-IDF / Arduino框架，提供：

- **模块驱动API**：注册新模块类型、读写传感器
- **AI接口API**：将模块数据发送给云端AI、接收AI指令
- **表情引擎API**：控制圆屏动画
- **音频API**：语音输入输出
- **OTA更新**：支持社区固件空中升级

> SDK文档：`docs/specs/firmware-sdk.md`（待编写）

## 10. 设计原则总结

1. **AI First：** 硬件为AI服务，不是反过来。每一个设计决策先问"AI需要这个吗？"
2. **温暖：** 这是一个有生命感的物体，不是一个电子产品。交互要有温度。
3. **克制：** 少即是多。不要堆功能，不要主动骚扰。安静的陪伴比喋喋不休更珍贵。
4. **可触：** 必须是物理的、可以摸到的、放在桌上的。不是又一个App。
5. **有趣：** 要让人微笑。连接新模块时的惊喜感、AI的俏皮话、意想不到的洞察。
6. **开放：** 一切开源。外壳、电路、固件、AI接口。社区的力量大于公司的力量。
7. **可制造：** 任何有3D打印机的人都能做出来。这不是概念设计，是可以今天就打印的东西。
