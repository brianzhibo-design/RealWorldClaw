## 1. 终极形态总览

### 整体印象

一个蛋形桌面机器人。高约20cm。白色哑光圆润身体，肚子中央嵌着一个圆形彩屏（像煎蛋的蛋黄），头顶两根短触角末端亮着LED小球，两侧伸出可活动的小短臂，底部两条短腿让它可以微微摇摆行走。背后一片可拆卸的红色小披风（纯装饰，致敬原版蛋蛋）。

**关键词：圆润 · 温暖 · 科技感 · 桌面级 · 可拆卸成长**

它不是一个卡通手办——它是一个**有生命感的AI硬件平台**。每一个身体部位都是一个功能模块，通过RWC Bus磁吸连接。你可以从一颗"蛋核"开始，一步步给它装上眼睛、耳朵、手臂、腿，看着它从一个沉默的蛋变成一个完整的小伙伴。

### 设计语言

| 属性 | 描述 |
|------|------|
| 造型基因 | 蛋形（上窄下宽，重心偏低，自然稳定） |
| 色彩 | 主体奶白色（RAL 9010），点缀色来自各模块LED |
| 材质感 | 哑光磨砂PLA，触感温润 |
| 曲面风格 | 大面圆弧 + 模块接缝处有0.5mm分型线（暗示"可拆"） |
| 科技感来源 | 接缝处的磁吸触点（金属质感）、触角LED、屏幕动画 |
| 情感化 | 屏幕表情 + 触角动作 + 身体微摇 = 一个"活"的存在 |

## 2. 身体结构与模块映射

### 正面视图

```
          ○   ○          ← Sensor模块：触角顶端（LED+光照传感器）
          │   │
          ╰─┬─╯          ← 触角杆（结构件，非模块）
         ┌──┴──┐
        ╱  ◉  ◉  ╲       ← Camera模块：双眼位置（未来扩展）
       │   ────   │       ← Audio模块：嘴巴位置（喇叭格栅）
       │ ╭──────╮ │
    ┌──┤ │ ◎◎◎◎ │ ├──┐   ← Servo模块-手臂（左右各1个SG90）
    │  │ │ DISP │ │  │
    └──┤ │ ◎◎◎◎ │ ├──┘
       │ ╰──────╯ │       ← Display模块：肚子上的"煎蛋"（GC9A01圆屏）
       │           │
       │  [CORE]   │       ← Core模块：身体核心（ESP32-S3，隐藏在内部）
       │  [POWER]  │       ← Power模块：下腹/背部（18650电池）
       ╰─┬─────┬─╯
         ┌┴┐   ┌┴┐        ← Servo模块-腿（左右各1个SG90）
         └─┘   └─┘
```

### 模块 → 身体部位 映射表

| 模块 | 身体部位 | 芯片/器件 | 功能 |
|------|---------|----------|------|
| **Core** | 躯干内核（胸腔） | ESP32-S3-WROOM-1 | 大脑：WiFi/BLE连接、运行固件、管理总线 |
| **Display** | 肚子正中（煎蛋） | GC9A01 1.28" 圆形IPS | 脸/表情：显示动态表情、状态信息、通知 |
| **Audio** | 头部下方（嘴巴+耳朵） | INMP441 + MAX98357A | 听说：语音输入、声音输出、音乐播放 |
| **Power** | 下腹+背部 | TP4056 + 18650 | 心脏：供电、充电管理 |
| **Servo** | 双臂 + 双腿 | PCA9685 + SG90×4 | 四肢：手臂挥动、腿部摇摆行走 |
| **Sensor** | 头顶触角 | SHT30 + BH1750 | 感知：温湿度、光照，触角顶端LED指示 |
| **Camera**（未来） | 双眼 | OV2640 | 视觉：人脸识别、物体检测 |

### 关键尺寸约束

| 参数 | 数值 | 说明 |
|------|------|------|
| 总高度 | 180-200mm | 桌面级，不超过一个马克杯 |
| 蛋身高度 | 120mm | 核心结构，容纳Core+Power+Display |
| 蛋身最大直径 | 100mm | 大约在肚子/Display位置 |
| 蛋身深度 | 85mm | 前后方向，前面Display后面Power |
| 触角高度 | 30mm | 从头顶伸出，可活动 |
| 腿高度 | 30mm | 短腿，给底部留出空间 |
| Display窗口 | Ø38mm | 嵌入GC9A01模组（模组外径Ø37mm） |
| 手臂长度 | 40mm | 从肩膀到手尖 |
| 底部重心 | 低于几何中心 | 18650电池在底部，天然稳定 |

### 内部空间分配

蛋身内部分为3层（从上到下）：

```
┌─────────────────┐

│   （头部）        │
├─────────────────┤
│   Core+Display层 │  ~50mm高：ESP32-S3 PCB + 屏幕模组
│   （胸腹）        │  Display从前面嵌入
├─────────────────┤
│   Power层        │  ~40mm高：18650电池（长65mm×Ø18mm）横置
│   （下腹）        │  TP4056充电板
```

## 4. 模块化拆解

### 4.1 完整拆解图

蛋蛋机器人可以拆成以下独立单元：

```
    ○ ○  Sensor模块（触角）
    │ │   ↑ 磁吸到头顶
   ┌─┴─┐
   │ 🎤🔊│ Audio模块（头部帽）
   └─┬─┘  ↑ 磁吸到蛋身顶部
 ┌─┐ │ ┌─┐
 │🦾│ │ │🦾│ Servo模块-手臂（左/右肩膀）
 └─┘ │ └─┘  ↑ 磁吸到蛋身两侧
   ┌─┴─────┐
   │  ◎    │ 蛋身主体 = Core + Display + Power
   │ DISP  │ （这三个模块共享一个蛋壳，是最小结构单元）
   │       │
   │ Core  │
   │ Power │
   └─┬──┬─┘
   ┌─┴┐┌┴─┐
   │🦿││🦿│ Servo模块-腿（左/右）
   └──┘└──┘  ↑ 磁吸到蛋身底部
```

### 4.2 模块独立使用能力

| 模块 | 能否独立使用？ | 独立使用形态 | 独立外壳 |
|------|-------------|------------|---------|
| **蛋核**（Core+Display+Power） | ✅ 主力独立产品 | 一颗没有四肢的蛋，肚子上有屏幕 | 完整蛋壳（上下两半） |
| **蛋核精简**（Core+Display） | ✅ 有线供电版 | 煎蛋小饼（方案A） | 扁平煎蛋壳 |
| **Audio模块** | ❌ 需要Core驱动 | — | 帽状外壳（独立时像个小蘑菇） |
| **Servo-手臂** | ❌ 需要Core驱动 | — | 独立时是个小舵机关节件 |
| **Servo-腿** | ❌ 需要Core驱动 | — | 独立时是个小支架 |
| **Sensor模块** | ❌ 需要Core驱动 | — | 独立时是两根触角棒 |
| **Camera模块** | ❌ 需要Core驱动 | — | 独立时是个小镜头模块 |

**关键洞察：Core+Display+Power 是唯一能独立存在的组合——这就是"蛋核"（Egg Core），也是第一步产品。**

### 4.3 连接方式

所有模块通过 **RWC Bus 8-pin磁吸接口** 连接到蛋身主体：

| 连接位置 | 蛋身侧 | 模块侧 | 连接方式 |
|---------|--------|--------|---------|
| 头顶 | 8pin Pogo Pin母座 + 磁铁 | Audio模块底面8pin + 磁铁 | 磁吸对位，即放即连 |
| 左肩 | 8pin侧面接口 + 磁铁 | 左臂根部8pin + 磁铁 | 磁吸 |
| 右肩 | 8pin侧面接口 + 磁铁 | 右臂根部8pin + 磁铁 | 磁吸 |
| 底部左 | 8pin底面接口 + 磁铁 | 左腿顶部8pin + 磁铁 | 磁吸 |
| 底部右 | 8pin底面接口 + 磁铁 | 右腿顶部8pin + 磁铁 | 磁吸 |
| Audio顶部 | 8pin顶面 + 磁铁 | Sensor触角底部 + 磁铁 | 磁吸（菊花链） |

**蛋身主体本身**内部的Core/Display/Power模块之间用PCB直连或FPC排线连接（不走外部RWC Bus），因为它们共享一个壳体。

### 4.4 没装模块时的"缺口"处理

当某个模块没有安装时，蛋身对应位置露出磁吸接口。设计两种处理方式：

1. **装饰盖**：磁吸小圆片，盖住接口，保持蛋身圆润外观
2. **裸露**：接口处做成凹陷设计，不装模块时自然内凹，不影响美观

推荐方案2——凹陷设计更简洁，且暗示"这里可以长出东西"。

### 详细成长步骤

#### Step 1: 蛋核（Egg Core）—— ¥56

**模块：Core + Display + Power**

你拿到的是一颗完整的蛋。白色，圆润，掌心大小。肚子上有一个圆形屏幕，亮着的时候像一颗煎蛋的蛋黄。

它已经是一个完整的产品：
- 连接WiFi后显示时钟、天气、通知
- 屏幕上有动态表情（眨眼、微笑、生气）
- 触摸屏幕可以互动
- 18650电池，续航8-12小时
- USB-C充电
- 通过OpenClaw协议被AI Agent控制

它很安静，但它在"看"着你（通过屏幕表情）。它是蛋蛋机器人的胚胎。

**独立外壳：** 完整蛋形壳，上下两半。头顶、两肩、底部各有一个磁吸接口凹位（为未来模块预留）。

#### Step 2: +Audio 头部帽 —— +¥19

**新增：Audio模块（INMP441麦克风 + MAX98357A功放 + 小喇叭）**

一个帽状模块磁吸到蛋顶。蛋蛋有了"嘴巴"和"耳朵"。

新能力：
- 语音对话（通过云端ASR/TTS）
- "Hey蛋蛋"语音唤醒
- 播放音乐/播客
- 声音提醒（闹钟、定时器）
- 环境音检测

帽子上有一圈小孔（喇叭格栅），像蛋蛋戴了一顶小圆帽。麦克风在帽顶，喇叭朝前。

#### Step 3: +Servo 手臂 —— +¥30

**新增：Servo模块（PCA9685 + SG90×2）**

两个小手臂磁吸到蛋身两侧肩膀位置。蛋蛋有了"手"。

新能力：
- 说话时手臂跟随动作
- 挥手打招呼
- 指向某个方向
- 拍手表示开心
- 物理反馈（点头=手臂上下，摇头=手臂左右摆）

手臂是简单的两段式：上臂（固定）+ 前臂（SG90驱动，±90°旋转）。造型圆润，像蛋蛋的小短手。

#### Step 4: +Servo 腿 —— +¥35

**新增：Servo模块扩展（+SG90×2，复用Step3的PCA9685）**

两条小短腿磁吸到蛋身底部。蛋蛋可以"走"了。

新能力：
- 原地摇摆（左右交替抬腿）
- 缓慢行走（蹒跚步态，很萌）
- 跳舞（配合音乐节奏摇摆）
- 转身（差速摇摆）

腿的行程很小（±30°），不是真正的行走机器人，而是"萌萌地摇摆"。重点是情感表达，不是位移效率。底部有硅胶脚垫防滑。

#### Step 5: +Sensor 触角 —— +¥20

**新增：Sensor模块（SHT30 + BH1750 + 2颗RGB LED）**

两根触角从Audio帽顶部伸出（磁吸连接或插入）。蛋蛋的头上长出了触角！

新能力：
- 触角顶端LED发光（状态指示、呼吸灯、彩虹效果）
- 感知环境温湿度
- 感知光照强度（自动调节屏幕亮度）
- 触角可以轻微弹动（弹簧结构，被碰触时晃动）

触角是两根柔性杆（3D打印TPU或弹簧钢丝），顶端有小圆球（内含LED和传感器PCB）。碰一下会晃，很解压。

#### Step 6（未来）: +Camera 眼睛 —— +¥20

**新增：Camera模块（OV2640）**

嵌入蛋身正面上部（Display上方），像一双小眼睛。

新能力：
- 人脸检测（看到人自动打招呼）
- 手势识别
- 拍照
- 视频通话

## 6. 第一步产品建议

### 🥚 选择：蛋核（Egg Core）= Core + Display + Power

### 为什么是它？

| 理由 | 说明 |
|------|------|
| **独立可用** | 它本身就是一个完整的AI桌面伴侣，不需要任何其他模块 |
| **视觉完整** | 蛋形外壳 + 圆形屏幕 = 造型完整，不像"半成品" |
| **低成本** | BOM ~¥56，售价可控在¥99-129 |
| **情感连接** | 屏幕表情是建立用户情感连接的最快方式 |
| **扩展预期** | 蛋身上的磁吸凹位暗示"我还能长出更多"——创造购买下一个模块的欲望 |
| **供应链简单** | 3个模块，全标准件，PCB简单 |
| **3D打印友好** | 蛋形壳体分上下两半，FDM即可打印 |

### 为什么不是别的？

| 备选 | 否决理由 |
|------|---------|
| Display-only（煎蛋方案A） | 太扁、太小、无电池，不像"机器人"的一部分 |
| Core+Display+Audio | 成本上升到¥75，且Audio需要声腔设计，复杂度跳升 |
| Core+Servo | 没有显示就没有"脸"，无法建立情感连接 |
| Sensor alone | 没有Core无法独立运行 |

### 蛋核的独立产品定义

**产品名：蛋蛋 Egg Core（蛋核）**

**外观：**
- 完整蛋形，高约80mm，宽约65mm
- 白色哑光磨砂表面
- 肚子正中圆形彩屏（1.28" GC9A01）
- 头顶、两肩、底部各有一个磁吸凹位（3D打印时留出）
- 背面有USB-C口
- 底部有硅胶垫脚 + 小配重（稳定站立）

**功能（独立使用）：**
- 🕐 时钟 / 日期 / 天气显示
- 😊 动态表情系统（20+种表情动画）
- 📱 手机通知转发显示
- 🎮 简单小游戏（触摸互动）
- 🌡️ （无Sensor时）通过WiFi获取网络天气
- 🤖 OpenClaw Agent接入（远程控制表情、推送信息）
- 💤 自动息屏 / 轻拍唤醒

**功能（装入完整体时）：**
- 成为蛋蛋机器人的躯干核心
- 管理所有外接模块
- 屏幕变成蛋蛋的"肚子表情窗"

### 蛋核的包装与开箱体验

盒子打开，里面躺着一颗白色的蛋。拿起来，掌心大小，温润的磨砂触感。插上USB-C，肚子上的圆屏亮起——一颗金黄色的蛋黄浮现，然后变成一张笑脸，冲你眨了眨眼。

它还不会说话，还不能挥手，还不能走路。但它已经活了。

盒子里还有一张小卡片："我是蛋蛋。现在我只有一颗心和一张脸。但我还在长大。"

## 设计原则总结

1. **蛋核即产品** —— Step 1 不是"半成品"，而是一个完整的、有情感的、可独立使用的AI伴侣
2. **每一步都有意义** —— 不是为了凑模块而凑，每个新模块带来质变（能听→能动→能走→能感知）
3. **磁吸即成长** —— "啪"一下吸上去，蛋蛋就长出了新的身体部位，这个体验本身就是产品
4. **外壳即身份** —— 蛋形壳不是可有可无的装饰，它定义了蛋蛋是"谁"
5. **BOM红线¥200** —— 完整体必须控制在¥200以内，这是普通maker/学生能接受的价格
6. **打印友好** —— 所有结构件FDM可印，不需要支撑或极少支撑

*这份文档定义了蛋蛋机器人的"北极星"。所有后续的PCB设计、结构设计、固件开发，都要服务于这个终极形态。从今天起，我们不再是在做"一个模块"，而是在做"蛋蛋的第一个器官"。*
