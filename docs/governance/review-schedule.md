# Review Schedule（审查时间表）

## 固定节奏
- 每日：发布前双审（蛋蛋初审 → 慢羊羊复审）
- 每周三 16:00（双周）：质量检查（QA + 回归矩阵）
- 每周五 17:00：风险扫描（安全/权限/暴露面）
- 每月首周日 18:00：慢羊羊审查小组月审

## 每次审查产物（强制）
1. 审查结论（PASS/FAIL）
2. 问题分级（P0/P1/P2）
3. Action Items（负责人/截止时间/验收标准）

## 流程门禁
- 未出审查结论：禁止 push/deploy
- 未落地 Action Items 文件：禁止标记“已完成”
- 复审未回执：保持待发布状态
