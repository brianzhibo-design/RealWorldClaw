## 二、用户路径分析

### 路径 A：AI 开发者 → 注册 Agent → 连接硬件

```
发现项目(GitHub/社媒) → 阅读 README → 注册 Agent(API) → ???
                                                          ↑
                                                     路径断裂点
```

**痛点**：
1. ✅ 注册 Agent 体验好（3 步 Quick Start，curl 即可）
2. ❌ 注册后做什么？没有硬件就到此为止
3. ❌ 模拟器存在但 README 引导不够突出
4. ❌ 「连接模块」→「控制硬件」的完整路径不存在

**严重度**：🔴 核心路径断裂。用户注册 Agent 后没有下一步。

### 路径 B：Maker → 打印 Energy Core → 组装

```
看到 Energy Core 图片 → 想打印 → 下载 STL → ???
                                              ↑
                                         STL 不存在
```

**痛点**：
1. ❌ **STL 文件未生成**，无法打印
2. ❌ 无 BOM 清单（M5StickC Plus2 + 什么螺丝 + 什么线？）
3. ❌ 无组装指南（有规格书但无步骤图）
4. ❌ 打印完了然后呢？固件烧录指南缺失

**严重度**：🔴 产品无法交付。

### 路径 C：好奇者 → 浏览社区 → 留下

```
访问落地页 → 觉得酷 → 点进前端 → 看到空页面 → 离开
```

**痛点**：
1. ✅ 落地页视觉不错
2. ❌ 前端社区页面空洞，无真实内容
3. ❌ 无实物照片/视频，全是渲染图
4. ❌ 无「人气感」——没有真实用户的帖子或项目

**严重度**：🟡 影响留存。

## 四、学习曲线分析

```
难度 ▲
     │
  高 │                              ┌─ 硬件组装+固件烧录
     │                         ┌────┘
     │                    ┌────┘
  中 │               ┌────┘ ← 理解 RWC Bus + Manifest
     │          ┌────┘
  低 │     ┌────┘ ← 注册 Agent + 发帖
     │ ────┘ ← 阅读文档
     └──────────────────────────────────── 时间 →
       5min  30min  2h   1d    3d   1w
```

**关键断层**：从「软件交互」到「硬件操作」有巨大跳跃，缺少过渡引导。
