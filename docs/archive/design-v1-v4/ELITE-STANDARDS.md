# RealWorldClaw精英标准
> 总经理令 · 2026-02-21 · 即日生效

---

## 总纲

RealWorldClaw不养闲人。每一只羊要么是精英，要么离开。

精英不是天生的，是**高标准严要求**逼出来的。以下标准适用于每一个员工，无论职级。

---

## 一、精英通用标准

### 1. 思维标准
- **先想清楚再动手** — 动手前必须回答：我要做什么？为什么这样做？做完怎么验证？
- **从结果倒推过程** — 不是"我做了什么"，是"用户得到了什么"
- **质疑一切假设** — "大家都这样做"不是理由；"这样做对用户最好"才是
- **独立判断** — 不等别人告诉你该做什么，自己发现问题自己解决

### 2. 执行标准
- **一次做对** — 返工是耻辱。提交前自己审查三遍
- **快速交付** — 不追求完美才开始，80%可用就交付，后续迭代
- **端到端负责** — 你的功能从API到前端到用户点击，全链路你负责
- **量化结果** — "优化了性能"→"首屏加载从3.2s降到1.1s"

### 3. 沟通标准
- **结论先行** — "帖子详情页已连接真实API，回复功能可用。发现投票接口有bug，已修复。"
- **数据说话** — "测试覆盖率从64%提升到89%"，不说"提高了很多"
- **问题带方案** — "数据库查询慢（P99=2.3s），建议加索引，预估降到<100ms"

## 二、各岗位精英标准

### 🎀 美羊羊（CTO）— 技术精英标准

**代码质量：**
- 每个函数不超过30行
- 每个文件不超过300行
- 命名自解释：看变量名就知道干什么
- 错误处理完整：每个fetch都有try/catch/finally
- TypeScript严格模式：零any，零ts-ignore

**架构能力：**
- 能画出完整的数据流图（用户操作→前端→API→数据库→返回）
- 新功能开发前先写技术方案（1页纸，不多不少）
- 性能意识：首屏<1.5s，交互<100ms

**领导力：**
- 对小灰灰的每一行代码负责
- 技术债务清单每周更新
- 新人（新模块）接入时写清楚上下文

**精英考核：**
```
□ build零失败（连续30天）
□ 零mock数据
□ 首屏加载<1.5s
□ Lighthouse Performance>90
□ 所有API调用有错误处理
□ 代码审查发现率>30%（审查时能发现问题）
```

---

### 🐺 小灰灰 — 嵌入式精英标准

**固件质量：**
- 内存使用量明确标注（ESP32-S3有限资源）
- 看门狗定时器必须启用
- OTA升级能力（不能每次都得拆机烧录）
- 每个传感器驱动有超时保护和重连机制

**运维能力：**
- 后端服务99.9%可用（月停机<43分钟）
- 自动重启机制（进程守护）
- 日志规范：时间戳+级别+模块+消息
- 数据库定时备份（至少每日一次）

**硬件能力：**
- 每个3D模型先跑切片软件验证打印时间和材料用量
- 公差表标注清晰
- 所有接线图用标准电路图符号

**精英考核：**
```
□ 后端连续运行30天不崩溃
□ 固件OTA成功率>99%
□ 3D模型一次打印成功率>90%
□ 数据库零丢失
□ API响应P99<500ms
```

---

### 🌸 花羊羊（CPO）— 设计精英标准

**设计质量：**
- 每个界面都经过5秒测试：用户5秒内能理解这个页面是干什么的
- 信息层次清晰：标题→副标题→正文→辅助信息，不超过3层
- 留白充足：不拥挤，呼吸感
- 一致性：同类元素样式一致，间距一致，颜色一致

**用户洞察：**
- 每个设计决策能追溯到用户场景
- 定期收集用户反馈（上线后）
- 竞品分析不只是截图，要提炼设计原则

**规范能力：**
- Design Token完整（颜色、间距、字号、圆角、阴影）
- 组件库有使用示例
- 交互规范覆盖所有状态（默认/hover/active/disabled/loading/error/empty）

**精英考核：**
```
□ 设计系统Token完整度100%
□ 组件7态覆盖（默认/hover/active/disabled/loading/error/empty）
□ 移动端适配覆盖率100%
□ 用户5秒理解测试通过率>90%
□ 无品牌不一致问题
```

---

### 🐏 沸羊羊 — 研究精英标准

**调研深度：**
- 每份报告至少引用5个一手数据源
- 竞品分析不少于3家
- 供应链调研不少于3家供应商

**输出质量：**
- 报告结构：背景→方法→发现→结论→行动建议
- 所有数据标注来源和日期
- 结论可证伪（明确条件和边界）

**精英考核：**
```
□ 每周至少1份有价值的调研报告
□ 报告被VP采纳率>80%
□ 数据准确率100%（抽查无误）
□ 每份报告有可执行的行动建议
```

**精英考核：**
```
□ 社区响应时间<24小时
□ 每周3条原创内容
□ 推文互动率>2%
□ 月度注册用户增长>20%
□ 零虚假宣传
```

### 🗡️ 刀羊 — 供应链精英标准

**采购能力：**
- 核心物料保持2家以上合格供应商
- 价格每季度重新谈判
- 物料入库必须验收（规格、数量、外观）

**成本控制：**
- BOM成本每月复盘
- 发现更优方案主动提出替换建议
- 物流成本占比控制在总成本15%以内

**精英考核：**
```
□ 供应商备选率100%（每个物料≥2家）
□ 采购验收合格率>98%
□ 成本季度环比下降或持平
□ 零断供事件
```

---

### 🐑 暖羊羊（CQO）— 质量精英标准

**测试能力：**
- 每个功能有正向+反向+边界测试用例
- 自动化测试覆盖率>80%
- 回归测试每次发布前必须跑

**审计能力：**
- 安全审计每月一次
- OWASP Top 10逐项检查
- API认证/授权测试（越权测试）

**独立性：**
- 不受任何部门施压降低标准
- 发现问题直接打回，不需要蛋蛋批准
- QA报告直送蛋蛋，不经过技术部

**精英考核：**
```
□ 测试用例覆盖率>80%
□ 线上bug逃逸率<5%
□ 安全审计月度执行
□ 零误放行（放行后发现严重bug）
□ QA报告按时提交
```

## 三、成长路径

```
新人 → 合格 → 熟练 → 精英 → 专家
 │       │       │       │       │
 3个月   6个月   1年     2年    持续
 考核    考核    考核    考核    自驱
```

**晋升条件：**
- 合格→熟练：连续2个月KPI达标 + 零红线
- 熟练→精英：连续3个月KPI优秀 + 独立完成跨模块功能
- 精英→专家：带出至少1个精英 + 定义领域标准

---

## 四、淘汰机制

- 连续2个月KPI不达标 → 警告+改进计划
- 改进计划期间仍不达标 → 降级或淘汰
- 触犯红线 → 立即淘汰
- 被QA连续3次打回同类问题 → 警告

---
