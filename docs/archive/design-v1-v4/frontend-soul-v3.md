# RealWorldClaw 前端灵魂 v3

> "帮你的AI长出身体"
> 设计哲学：温暖 · 克制 · 可触 · 有趣

## 一、核心体验流

### 第一次来到平台，看到什么？

不是仪表盘。不是注册表单。

用户看到的是一个**安静的、微微呼吸的3D身体轮廓**——半透明，像一个等待被填满的容器。旁边一行字：

> "你的AI还没有身体。要不要帮它长一个？"

一个按钮：**"开始创造"**

没有导航栏。没有功能介绍。只有这个画面和这个问题。

### 什么让他们留下？

**第一个器官的诞生。** 用户选择第一个想要制造的部件（比如一只机械手），系统会用一段简短的动画展示：这个部件从数字模型变成实体的过程——切片、路径规划、打印机启动。几秒钟内，用户理解了：**这不是虚拟的，这是真的会被造出来的东西。**

然后身体轮廓上，那个部位亮了起来。

### 什么让他们兴奋？

**身体在成长。** 每完成一个部件的打印，3D身体模型上对应的器官从半透明变为实体。用户看到自己的AI一点点"长出"身体。这个成长感是持续的多巴胺。

进度不是百分比数字，是**视觉上的生命力增长**。

### 什么让他们传播？

**"看，我的AI长出手了。"** 每个里程碑（第一个器官、半身、完整体）都生成一张可分享的卡片——3D渲染的身体状态，配上一句生成的文案。用户会截图发朋友圈，因为这看起来不像在用一个SaaS产品，像在养一个生命。

## 三、核心页面设计

### 1. 生命体 `/body` — "这是你的AI"

**情感定位：** 自豪。像看着自己的孩子在成长。

**核心交互：**
- 中央是一个可旋转的3D身体模型
- 已完成的器官是实体、有质感、微微发光
- 未完成的是半透明轮廓，点击可进入器官工坊
- 鼠标悬停某个器官，浮出它的状态：制造中/已完成/待设计

**视觉想象：**
深色背景。身体模型悬浮在中央，底部有淡淡的光晕。实体部分是温暖的米白色，带有3D打印层纹的微妙质感——提醒用户这是真实的、被制造出来的。半透明部分用极淡的蓝色描边。整个画面安静、有呼吸感。

左下角小字显示：`3/12 器官已培育 · 距离完整体还需 9 个部件`

### 2. 器官工坊 `/workshop` — "为AI选择新器官"

**情感定位：** 探索的快乐。像在给角色选装备，但这次是真的。

**核心交互：**
- 器官卡片网格，每张卡片是一个3D部件的缩略渲染
- 点击展开详情：尺寸、材料、预估培育时间、价格
- "请求培育"按钮（= 下单）
- 支持上传自定义STL模型："带来你自己的器官"

**视觉想象：**
卡片采用圆角、微投影、悬停时轻微上浮。每张卡片底部一条颜色条表示复杂度（绿→黄→红）。顶部筛选不叫"分类"，叫"身体区域"：头部、躯干、四肢、感知器官。

### 3. 培育舱 `/incubator` — "你的器官正在成长"

**情感定位：** 期待与安心。像看着培养皿里的细胞在分裂。

**核心交互：**
- 每个进行中的器官是一个"培育卡片"
- 卡片内：实时进度可视化（不是进度条，是器官模型逐层"生长"的动画）
- 预计完成时间、当前阶段（切片中/打印中/后处理/质检/发货）
- 点击展开可看到打印机实时状态

**视觉想象：**
卡片有一层微微脉动的边框光晕，表示"活着、在进行"。打印中的器官显示一个简化的逐层堆叠动画。已完成的卡片边框变为稳定的绿色光晕，文案变为"器官已就绪，等待归体"。

### 4. 制造基地 `/foundry` — "这里是器官被铸造的地方"

**情感定位：** 信任。用户知道背后有真实的制造能力。

**核心交互：**
- 打印机群以"制造单元"卡片展示
- 每个单元显示：名称、状态（空闲/制造中/维护）、正在培育的器官
- 管理员可分配任务、调整队列
- 普通用户看到的是简化版：几台机器在工作，一切正常

**视觉想象：**
工业感但不冰冷。深灰背景，机器卡片有微妙的金属质感。运行中的机器卡片有一个小小的脉动点。整体像一个安静高效的实验室监控台。

### 5. 神经中枢 `/nerve-center` — "让AI连接物理世界"

**情感定位：** 智能感、连接感。"我的AI真的在控制东西了。"

**核心交互：**
- API密钥管理（包装为"神经通路"）
- Webhook配置（包装为"感知回路"）
- 与AI Agent的集成指引
- 实时事件流：显示AI最近的"感知"和"动作"

**视觉想象：**
左侧是连接列表，右侧是一个实时滚动的事件流，像神经信号在传递。每条事件是一个小气泡，从左到右流动。配色用冷蓝色调，暗示数据和智能。

### 6. 记忆库 `/memory` — "每一次创造都值得被记住"

**情感定位：** 回顾和意义感。

**核心交互：**
- 时间线展示所有历史：器官完成、里程碑达成
- 每个事件可展开查看详情
- 可分享的里程碑卡片

**视觉想象：**
纵向时间线，左侧是时间，右侧是事件卡片。卡片内有器官的3D缩略图和简短描述。最新的在顶部，向下滚动像在翻阅一本成长日记。

## 四、技术可行性与优先级

### Three.js 3D 可视化（核心差异化）

| 功能 | 复杂度 | 优先级 |
|------|--------|--------|
| 生命体页面的3D身体模型 | 高 | **P0** — 这是灵魂 |
| 器官卡片的3D缩略渲染 | 中 | P1 |
| 培育舱的逐层生长动画 | 中 | P1 |
| 首页的呼吸动画 | 低 | P0 |

**技术方案：**
- 使用 React Three Fiber（R3F）集成到 Next.js
- 身体模型：预制的分段式GLTF模型，每个器官独立mesh，通过材质切换表现状态
- 逐层生长：Shader动画，用clip plane从下往上逐渐显示模型
- 卡片缩略图：可用静态预渲染图降级，不必每张都实时3D

### 静态页面足够的部分

| 页面/功能 | 说明 |
|-----------|------|
| 器官工坊的列表/筛选 | 标准卡片网格，3D缩略图用静态图即可 |
| 制造基地 | 卡片式布局，状态指示器用CSS动画 |
| 神经中枢 | 表单+事件流，纯2D |
| 记忆库 | 时间线组件，纯2D |

### 优先级排序（开发阶段）

**Phase 1 — 灵魂上线（2周）**
1. 首页（呼吸动画 + CTA）
2. 生命体页面（3D身体模型，基础交互）
3. 器官工坊（卡片列表 + 下单流程）
4. 培育舱（订单状态，先用2D）

**Phase 2 — 体验深化（2周）**
5. 培育舱3D生长动画
6. 器官卡片3D缩略图
7. 里程碑分享卡片
8. 制造基地

**Phase 3 — 神经连接（2周）**
9. 神经中枢
10. 记忆库
11. 全局动效打磨

## 六、设计原则备忘

1. **温暖** — 不是赛博朋克，不是冰冷科技。色调偏暖，圆角，有呼吸感。
2. **克制** — 每个页面只做一件事。不堆功能，不堆文字。留白是信任。
3. **可触** — 时刻提醒用户：这不是虚拟的。3D打印的层纹质感、真实的机器状态、实物照片。
4. **有趣** — 微交互、小彩蛋。第一个器官完成时的庆祝动画。里程碑的文案要有人味。
