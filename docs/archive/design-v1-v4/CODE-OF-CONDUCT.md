# RealWorldClaw行为规范
> 适用于所有员工和外部贡献者

---

## 核心价值观

### 诚实 (Integrity)
- 数据真实，不伪造
- 进度真实，不夸大
- 问题真实，不隐瞒
- 能力真实，不虚报

### 责任 (Ownership)
- 自己的代码自己负责
- 自己的任务自己跟进
- 出了问题先修复，再分析原因
- 不推诿，不甩锅

### 卓越 (Excellence)
- 不满足于"能用"，追求"好用"
- 不满足于"完成"，追求"精品"
- 每次迭代都比上次好
- 向行业最高标准看齐

### 协作 (Collaboration)
- 主动分享，帮助同事
- 对事不对人，就事论事
- 接受批评，快速改进
- 尊重每个岗位的专业性

## 日常行为规范

### 沟通
1. **准时** — 晨会准时参加，信息及时回复
2. **清晰** — 结论先行，数据说话，避免模糊表述
3. **建设性** — 提问题时带解决方案
4. **尊重** — 不打断，不贬低，就事论事

### 工作
1. **计划先行** — 动手前想清楚要做什么、怎么验证
2. **及时同步** — 遇到阻塞立即上报，不自己扛
3. **完整交付** — 提交前自检清单全部通过
4. **知识沉淀** — 解决的问题写成文档

### 代码
1. **自解释** — 命名清晰，逻辑简洁
2. **有注释** — 解释"为什么"，不只是"是什么"
3. **有测试** — 新功能必须有对应测试
4. **有文档** — API变更必须更新文档

### 安全
1. **零信任** — 所有输入都需验证
2. **最小权限** — 只请求需要的权限
3. **密钥管理** — API密钥不入代码库
4. **及时修复** — 安全漏洞优先级最高

## 跨部门协作规范

### 需求提出
```
需求标题：[简洁描述]
提出人：[姓名/部门]
优先级：[P0/P1/P2/P3]
背景：[为什么需要这个]
期望结果：[具体可衡量的结果]
截止时间：[具体日期]
```

### 问题上报
```
问题描述：[具体现象]
影响范围：[哪些功能/用户受影响]
重现步骤：[1. 2. 3.]
期望行为：[应该是什么样]
实际行为：[实际是什么样]
我的建议：[如何修复]
```

### 任务交接
```
任务名称：[名称]
当前状态：[完成/进行中/阻塞]
已完成内容：[具体列表]
待完成内容：[具体列表]
注意事项：[坑/风险/依赖]
相关文档：[链接]
```

## 对外行为规范

### 社交媒体
- 不泄露未发布的产品信息
- 不贬低竞品
- 不承诺未实现的功能
- 保持专业、友好的语气

### 开源社区
- 及时回复Issue和PR
- 对贡献者保持耐心和感谢
- 代码审查给出具体、建设性的反馈
- 遵循开源协议（Apache 2.0 / ELv2）

### 合作伙伴
- 所有对外合作承诺需蛋蛋审批
- 商务邮件使用公司模板
- 守时、守信、守约

## 冲突解决

1. **当事人直接沟通** — 大部分问题可以直接解决
2. **VP介入调解** — 沟通无果时由VP调解
3. **蛋蛋裁决** — VP也解决不了时蛋蛋最终裁决
4. **大人终审** — 涉及战略方向由大人决定

**原则：对事不对人，结果导向，快速解决。**

---

## 违规处理

| 违规类型 | 处理方式 |
|----------|----------|
| 轻微违规（迟到、格式不规范） | 口头提醒 |
| 一般违规（延迟交付、汇报不实） | 书面警告 |
| 严重违规（红线行为） | 立即开除 |
| 累计3次书面警告 | 开除 |
