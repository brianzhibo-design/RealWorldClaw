## 🖨️ 推荐打印参数

### 通用设置（PLA）

| 参数 | 值 | 说明 |
|------|-----|------|
| 材料 | PLA | 首次测试优先 PLA，稳定易打 |
| 喷嘴温度 | 200-210°C | 根据品牌微调 |
| 热床温度 | 60°C | |
| 层高 | 0.2mm | 平衡速度和质量 |
| 壁数 | 3 (≈1.2mm) | |
| 填充率 | 20% | 网格或回转填充 |
| 打印速度 | 50-60mm/s | 外壁降到 30mm/s |
| 支撑 | 仅上壳内部极少量 | Peace手势不需要支撑 |
| 裙边/Brim | 5mm brim | 底座较大，建议加 brim |
| 冷却 | 100%（第3层后） | |
| Z-seam | 后方对齐 | 减少正面可见接缝 |

### 不同打印机配置参考

#### Bambu Lab A1 / P1S / X1C
- 使用 Bambu Studio 默认 PLA 配置即可
- 速度可提升到 100mm/s（X1C）
- AMS 可用多色：底座用深灰，身体用蟹橙

#### Creality Ender-3 / K1
- Ender-3：速度保守 40-50mm/s，确保调平
- K1：可用 80mm/s，开 input shaping

#### Prusa MK3S+ / MK4
- 使用 PrusaSlicer 0.2mm QUALITY 配置
- 开启 Arachne 变宽挤出

#### Voron / Klipper 系
- 建议 pressure advance 校准后再打
- 速度可上 100mm/s+，加速度 3000+

### 分件打印策略

建议**分三次打印**，逐步验证：

1. **第一打：底座 (base)** — 约 1h，验证尺寸精度和卡扣
2. **第二打：上壳 (body)** — 约 1.5h，验证配合和开孔
3. **第三打：爪子 (claw)** — 约 1h，验证手指细节

在 OpenSCAD 中切换 `render_part` 参数：
```
render_part = "base"   // 只渲染底座
render_part = "body"   // 只渲染上壳
render_part = "claw"   // 只渲染爪子
```

## 🔧 OpenSCAD 导出 STL 步骤

### 1. 打开文件
- 启动 OpenSCAD
- `File → Open` → 选择 `models/rwc-one-peace.scad`
- 等待加载（文件含多个模块）

### 2. 预览检查（F5）
- 按 `F5` 预览整体组装效果
- 确认三部分都正常显示：底座（深灰）、身体（橙色）、爪子（橙色）
- 用鼠标旋转检查各角度

### 3. 选择要导出的部件
- 在编辑器底部找到 `render_part` 参数
- 修改为需要的部件：`"base"` / `"body"` / `"claw"`
- 或使用右侧 Customizer 面板（`Window → Customizer`）

### 4. 渲染（F6）
- 按 `F6` 进行完整渲染（CGAL）
- ⚠️ 渲染可能需要 30秒 - 3分钟，取决于电脑性能
- 底部状态栏显示进度

### 5. 导出 STL
- 渲染完成后：`File → Export → Export as STL`
- 命名建议：`rwc-one-peace-base.stl`、`rwc-one-peace-body.stl`、`rwc-one-peace-claw.stl`
- 保存到 `models/stl/` 目录

### 6. 验证 STL
- 在切片软件中打开 STL
- 检查是否有非流形错误（Cura/PrusaSlicer 会自动提示）
- 确认尺寸正确（底座直径应为 70mm）

## ⚠️ 打印过程中注意事项

### 第一层
- **最关键！** 观察第一层是否均匀粘附
- 如果翘边 → 重新调平 / 加胶棒 / 提高热床温度
- 前 5 分钟不要离开

### 中间阶段
- 注意听异常声音（堵头、打滑）
- 检查是否有翘曲或层裂
- 每 30 分钟检查一次（或用摄像头监控）

### 细节部分
- 爪子的手指较细（12mm 直径），注意散热冷却
- 卡扣凸起很小（1.2mm），确保精度
- 通风格栅（2mm 宽槽）需要足够冷却

### 紧急情况
- 如果严重偏移/翘边 → 立即停止，不要浪费耗材
- 如果轻微问题 → 记录下来，打印完后评估
- 堵头 → 停止，清理喷嘴，重新开始

## 🔥 常见问题和解决方案

### 问题1：底座翘边
- **原因**：热床温度不够 / 平台不平 / 无 brim
- **方案**：热床 65°C + 胶棒 + 5mm brim

### 问题2：卡扣太紧/太松
- **太紧**：在 OpenSCAD 中增大间隙 → 修改 `snap_depth` 从 1.2 到 1.0
- **太松**：减小间隙 → `snap_depth` 从 1.2 到 1.4
- **快速修**：砂纸打磨（太紧）或加一小点热熔胶（太松）

### 问题3：OLED 开孔尺寸不对
- **偏小**：用锉刀扩孔；或修改 `oled_diameter` +0.5mm 重打
- **偏大**：用少量热熔胶固定屏幕

### 问题4：手指断裂
- **原因**：PLA脆 + 冷却不足
- **方案**：降速到 30mm/s + 100% 冷却 + 考虑 PETG

### 问题5：通风格栅打不出来
- **原因**：2mm 槽太细，桥接不好
- **方案**：降速 + 提高冷却 + 可以适当放大 `vent_width` 到 2.5mm

### 问题6：渲染错误 / STL 非流形
- **方案**：确保 OpenSCAD 用最新版；`$fn=64` 已足够；如有问题用 Meshmixer 修复 STL

### 问题7：上壳和底座对不齐
- **原因**：收缩不均
- **方案**：确保两部分同一打印机、同一卷料、同一参数打印

## 🎯 测试目标

第一次打印测试的核心目标：

1. **验证 STL 可打印** — OpenSCAD → STL → G-code 流程通畅
2. **验证尺寸精度** — 关键配合尺寸在公差范围内
3. **验证卡扣设计** — 上下壳能可靠扣合/拆卸
4. **验证开孔尺寸** — OLED、USB-C、DHT22、按钮孔位正确
5. **记录改进项** — 为 v1.1 收集反馈

不追求完美，追求**可量化的数据**。加油！🦀✌️
