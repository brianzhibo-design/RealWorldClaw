# RealWorldClaw 平台规划 v1.0

> 让任何人的任何想法变成实物

## 一、我们到底在做什么

一个连接"有想法的人"和"有机器的人"的开放网络。

- **不是3D打印服务商**（我们不拥有任何机器）
- **不是电商平台**（我们不卖东西）  
- **是基础设施**（像GitHub连接代码和开发者，我们连接设计和制造者）

核心交付：**你上传一个设计文件，我们帮你找到最合适的制造者，把它变成实物寄到你手上。**

## 二、用户是谁，他们要什么

### 用户A：有想法/有文件的人
- "我设计了一个手机支架，谁能帮我打印？"
- "我需要10个定制零件，附近谁能做？"
- 需要：上传文件 → 看到报价 → 选择制造者 → 收到实物

### 用户B：有机器的人（制造者/Maker）
- "我有一台闲置的3D打印机，想接单"
- "我是CNC工厂，想找更多订单"
- 需要：注册设备 → 看到附近订单 → 接单 → 制造 → 交付 → 获得收入和信誉

### 用户C：开发者
- "我想给我的AI/应用接入制造能力"
- 需要：API文档、SDK、节点协议

## 三、核心用户流程

```
用户A的旅程：
  打开首页 → 看到"上传你的设计" → 上传STL文件
  → 选择材料/数量/期望时间 → 系统匹配附近制造者
  → 看到匹配结果（匿名，只显示能力/评分/距离）
  → 选择一个 → 制造者接单 → 跟踪进度 → 收到实物 → 评价

用户B的旅程：
  打开首页 → 看到"注册你的机器" → 填设备信息
  → 设备出现在地图上 → 收到匹配的订单通知
  → 查看订单详情 → 接单 → 制造 → 上传完成照片
  → 平台安排物流 → 获得收入和评分
```

## 四、平台功能模块

### Phase 1 — MVP（3月8日前）

#### 1. 首页
**不是空地图。** 首页是整个平台的门面。

结构：
- **Hero区**：一句话说清价值 + 两个大按钮
  - "我要制造" → 上传设计流程
  - "我有机器" → 注册节点
- **地图区**：世界地图，展示所有在线节点（作为社会证明，不是主功能）
- **How it Works**：3步说明
- **社区精选**：最新分享的设计 / 最新制造成功案例
- **数据统计**：节点数、已完成订单数、覆盖国家数（真实数据，不造假）

#### 2. 订单系统（核心）
- `/submit` — 上传设计文件（STL/OBJ/STEP）
  - 选材料、数量、颜色
  - 自动估算尺寸和打印时间
  - 选择：自动匹配 / 手动选制造者
- `/orders` — 我的订单列表
- `/orders/:id` — 订单详情、进度跟踪、消息沟通

#### 3. 节点系统
- `/register-node` — 注册制造设备
  - 设备类型、构建体积、支持材料
  - 大致位置（自动模糊化）
  - 能力标签
- `/my-nodes` — 管理我的设备
- `/map` — 独立地图页（可筛选、搜索）

#### 4. 社区（轻量版）
- `/community` — 设计分享 + 讨论
  - 分享设计文件（公开的，任何人可以"帮我制造这个"）
  - 展示制造成果（照片）
  - 简单的评论/讨论
- 这是平台活力的来源——没有社区，平台就是冷冰冰的工具

#### 5. 用户系统
- `/auth/login` `/auth/register`
- 用户资料页
- 信誉系统（完成率、评分）

#### 6. 通知系统
- 新订单匹配通知（给制造者）
- 订单状态更新（给用户）
- 社区回复通知

### Phase 2 — 增长期（3月后）
- 制造者评级和排行
- 设计文件在线预览（3D viewer）
- 自动报价系统
- 多语言完善
- 移动端优化
- 开发者API和SDK

### Phase 3 — 生态期
- 插件系统（第三方集成）
- 机器自动接单（API驱动）
- 更多制造类型（CNC、注塑、PCB）
- AI辅助设计优化（"你的设计可能打印失败，建议修改..."）

## 六、技术架构

```
realworldclaw.com (Landing/Marketing)
  ↓
app.realworldclaw.com (Main Application)
  ├── / (首页: hero + map + community)
  ├── /submit (上传设计)
  ├── /orders (订单管理)
  ├── /map (全屏地图)
  ├── /community (社区)
  ├── /register-node (注册设备)
  ├── /my-nodes (管理设备)
  ├── /auth/* (认证)
  └── /settings (设置)
  
api.realworldclaw.com (API)
  ├── /auth/* (认证)
  ├── /nodes/* (节点管理)
  ├── /orders/* (订单系统)
  ├── /files/* (文件上传/管理)
  ├── /community/* (社区帖子)
  └── /notifications/* (通知)

数据库: SQLite → PostgreSQL (生产)
文件存储: S3/R2 (设计文件)
实时: WebSocket (订单状态、通知)
```

## 七、MVP开发优先级

按用户价值排序，不是按技术难度：

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | 文件上传 + 自动匹配 | 核心价值，没有这个平台没意义 |
| P0 | 制造者接单界面 | 双边市场，两边都要有 |
| P1 | 首页重做 | 门面，决定第一印象 |
| P1 | 社区基础版 | 平台活力来源 |
| P1 | 订单跟踪 | 用户体验闭环 |
| P2 | 通知系统 | 让制造者知道有新单 |
| P2 | 信誉系统 | 信任基础 |
| P3 | 3D预览 | 好用但不紧急 |
| P3 | API文档完善 | 开发者生态 |

## 八、当前状态 vs 目标

| 模块 | 当前 | 目标 |
|------|------|------|
| 首页 | 空地图，不知道干什么 | Hero + 流程 + 地图 + 社区精选 |
| 上传设计 | 无 | 完整的文件上传 + 匹配流程 |
| 订单 | 空壳，连API没内容 | 完整订单生命周期 |
| 节点注册 | 有表单但不完善 | 完整注册 + 设备管理 |
| 社区 | 无 | 设计分享 + 讨论 + 成果展示 |
| 地图 | 能显示节点，但只有1个 | 全屏可交互，点击查看详情 |
| 通知 | 无 | 邮件/站内通知 |
| 用户资料 | 基础登录 | 完整资料 + 信誉 |

## 九、不做什么（边界）

- ❌ 不做支付（MVP阶段制造者和用户自行协商）
- ❌ 不做物流（用户和制造者自行安排）
- ❌ 不控制机器（只做匹配，不远程操控）
- ❌ 不做内容审核（信任社区自治）
- ❌ 不收费（零抽佣）
